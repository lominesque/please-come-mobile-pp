Privacy Policy

Effective May 22, 2026

Data Collected

  • Student ID — entered by the user, stored locally on-device in AsyncStorage. Never sent to any server.
  • Device UUID — auto-generated on first launch, stored in the iOS Keychain. Used only for local admin-unlock code verification. Never transmitted.

Data Broadcast

The student ID is advertised over Bluetooth Low Energy as a local BLE device name, visible to nearby BLE scanners operated by the institution. It is not transmitted over the internet.

Data NOT Collected

  • No account, login, or password.
  • No location data — the NSLocationWhenInUseUsageDescription key exists only because iOS requires it for Bluetooth. The app never calls the location API.
  • No analytics, crash reporting, or third-party SDKs.
  • No data is ever sent to any remote server.

Data Retention

Student ID and device UUID persist on-device until the device lock is cleared by an admin. The Keychain UUID survives app deletion on iOS.

Third Parties

✓ No third-party SDKs ✓ No network calls ✓ No backend
